# 民商事司法协助系统登录认证 Feature List

## 模块1：账号登录

| 功能编号 | 功能名称 | 功能描述 | 优先级 | 备注 |
|---------|---------|---------|--------|------|
| AUTH-01 | 邮箱账号输入 | 用户输入注册邮箱，系统校验邮箱格式，用于识别登录主体 | P0 | 首版以邮箱作为账号标识 |
| AUTH-02 | 密码输入 | 用户输入密码，系统校验非空并在后端完成密码正确性验证 | P0 | 前端不保存明文密码 |
| AUTH-03 | 图片验证码输入 | 用户输入图片验证码，系统校验非空并在后端校验验证码正确性 | P0 | 用于降低暴力破解风险 |
| AUTH-04 | 登录基础校验 | 用户点击 LOG IN 后，系统依次校验邮箱、密码、图片验证码，通过后进入邮箱验证码页 | P0 | 任一校验失败停留当前页并提示 |
| AUTH-05 | 找回密码入口 | 用户可点击 Forgot Password? 进入找回密码流程 | P1 | 当前原型仅保留入口 |
| AUTH-06 | 注册入口 | 未有账号用户可点击 SIGN UP 进入注册流程 | P1 | 当前原型仅保留入口 |
| AUTH-07 | 多语言入口 | 用户可在 English、Русский、Français 间切换语言 | P2 | 当前原型仅做视觉展示 |

## 模块2：邮箱二次验证码

| 功能编号 | 功能名称 | 功能描述 | 优先级 | 备注 |
|---------|---------|---------|--------|------|
| MFA-01 | 发送邮箱验证码 | 登录基础校验通过后，系统向当前邮箱发送 6 位验证码 | P0 | 页面展示脱敏后的发送目标邮箱 |
| MFA-02 | 发送按钮倒计时 | 邮箱验证码发送后，发送按钮进入 60 秒倒计时，倒计时结束后允许重新发送 | P0 | 防止频繁发送邮件 |
| MFA-03 | 验证码有效期 | 邮箱验证码自发送起 5 分钟内有效，超时后不可继续使用 | P0 | 超时需重新发送验证码 |
| MFA-04 | 邮箱验证码输入 | 用户输入邮箱验证码，点击校验并登录 | P0 | 验证码建议限制 6 位数字 |
| MFA-05 | 验证失败提示 | 用户输入错误、过期或已使用验证码时，系统展示明确错误提示 | P0 | 不暴露过多安全细节 |
| MFA-06 | 一次性使用 | 已校验通过的邮箱验证码立即标记为已使用，不允许二次使用 | P0 | 后端需保证原子性和幂等 |
| MFA-07 | 重新发送验证码 | 用户可在 60 秒后重新发送验证码，新验证码应使旧验证码失效或按策略共存 | P1 | 建议新验证码使上一条未用验证码失效 |
| MFA-08 | 邮箱脱敏展示 | 页面展示验证码发送邮箱时，对邮箱用户名部分脱敏，域名部分保留 | P0 | 例如 `alice.wang@example.com` 展示为 `ali***ng@example.com` |

## 模块3：系统首页跳转

| 功能编号 | 功能名称 | 功能描述 | 优先级 | 备注 |
|---------|---------|---------|--------|------|
| HOME-01 | 登录成功进入系统首页 | 用户邮箱验证码校验成功后，系统建立登录态并跳转系统首页 | P0 | 原型以 SYSTEM HOME 占位 |
| HOME-02 | 登录成功反馈 | 系统在跳转前或首页展示登录成功状态 | P1 | 生产环境可直接跳转，不必停留 |
| HOME-03 | 会话状态保持 | 登录成功后系统保存安全会话，用于后续访问鉴权 | P0 | 建议 HttpOnly Secure Cookie |

## 模块4：安全与审计

| 功能编号 | 功能名称 | 功能描述 | 优先级 | 备注 |
|---------|---------|---------|--------|------|
| SEC-01 | 登录失败限制 | 系统对连续登录失败、验证码失败、频繁发送验证码进行限流 | P0 | 按账号、IP、设备多维度控制 |
| SEC-02 | 验证码安全存储 | 系统只保存验证码哈希、有效期、使用状态，不保存可逆明文 | P0 | 前端原型为演示显示验证码，生产不可显示 |
| SEC-03 | 操作日志 | 系统记录登录尝试、验证码发送、验证码校验、登录成功/失败日志 | P0 | 用于安全审计和问题追溯 |
| SEC-04 | 风险提示与告警 | 异常登录、验证码爆破、邮件发送异常时产生安全告警 | P1 | 可接入风控或日志平台 |

## 完整性检查

| 检查项 | 结论 | 说明 |
|-------|------|------|
| 登录是否有找回密码 | 已覆盖 | 当前原型保留入口，后续需补充完整找回密码流程 |
| 登录是否有注册 | 已覆盖 | 当前原型保留入口，后续需补充注册协议、邮箱验证、账号激活 |
| 重复提交是否处理 | 部分覆盖 | 发送验证码有 60 秒倒计时；登录与校验按钮生产需增加 loading 和防重复提交 |
| 验证码过期是否处理 | 已覆盖 | 邮箱验证码 5 分钟有效 |
| 验证码二次使用是否处理 | 已覆盖 | 已通过验证码不可再次使用 |
| 失败次数限制是否处理 | 已纳入 PRD | PRD 中纳入安全要求，原型暂未实现 |
| 操作日志是否处理 | 已纳入 PRD | PRD 中纳入非功能需求 |
